## Codebase Patterns

- Django project package is `memorybank` (at `backend/memorybank/`), app is `core` (at `backend/core/`)
- DJANGO_SETTINGS_MODULE = `memorybank.settings`
- DB path configurable via `SQLITE_PATH` env var; media path via `MEDIA_ROOT` env var
- Run Django commands from `backend/` directory: `cd backend && python manage.py ...`
- Run tests: `cd backend && python manage.py test core`
- Custom User model: `core.User` extends `AbstractUser` with `display_name` and `avatar`; `AUTH_USER_MODEL = 'core.User'` in settings
- Balance computed on-the-fly via `_compute_balances()` in views.py — no stored state
- Soft delete pattern: `ExpenseManager` filters `is_deleted=False`; `Expense.all_objects` for unfiltered
- ExpenseSerializer handles split logic: `split_data` (write-only) for custom splits, auto equal splits when omitted
- AdminWritePermission: reads for all auth users, writes for staff only
- ApiKeyAuthentication via `X-API-Key` header in `core/authentication.py`
- prd.json branchName drives which git branch to work on; create from main if missing
- No code volume mount in docker-compose — must `docker compose up --build` to pick up code changes
- Fixture at `core/fixtures/test_data.json`; password for all fixture users: `password123`
- Load fixture: `python manage.py loaddata test_data` (6 objects: 2 users, 1 currency, 1 category, 1 group type, 1 group)
- Fixture PKs: admin=1 (staff), alice=2 (regular), currency=1 (USD), category=1 (Food), grouptype=1 (Home), group=1
